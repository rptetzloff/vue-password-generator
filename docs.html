Password Generator — Reference

This app generates passwords entirely in your browser. Nothing leaves your device — no network requests are made, no passwords are logged or stored on any server. Your settings and generation history are saved only in your browser's local storage.

Seven generator types are available, each optimized for a different use case. Click any tile to jump to its documentation.

Simple

Generates a random string of characters from selected character sets.

Example output
mX7!vQ#nR2@kLpT9&wZ
Options
Password Length 6 – 128
Total number of characters in the generated password. The slider controls the exact count.
Lowercase letters
Includes a – z in the character pool when checked.
Uppercase letters
Includes A – Z in the character pool when checked.
Numbers
Includes 0 – 9 in the character pool when checked.
Symbols
Includes the full symbol set ! # $ % & ( ) * + , - . / : ; < = > ? @ [ ] ^ _ ` { | } ~ when checked. At least one character type must remain selected.

Each character is drawn independently and uniformly at random from the combined pool. No placement guarantees are made — to enforce minimums per type, use the Advanced generator.

Advanced

Like Simple, but with per-character-type minimum and maximum counts, plus a customizable symbol set.

Example output (length 20, ≥2 of each type)
Kp3!mW#8vL$2rN@5qX&1
Options
Password Length 6 – 128
Total number of characters. Must be reachable by the sum of all minimums and not exceed the sum of all maximums.
Lowercase Letters
Min / Max
Min — at least this many lowercase letters will appear.
Max — no more than this many lowercase letters will appear.
Set Min to 0 to make lowercase optional.
Uppercase Letters
Min / Max
Same min/max controls for A – Z.
Numbers
Min / Max
Same min/max controls for 0 – 9.
Symbols
Min / Max + Symbol Set
Same min/max controls for symbols. Below the sliders, individual symbol chips let you choose exactly which symbols are included in the pool:

All — activate every symbol.
Common — activate a curated subset: ! @ # $ % & * - _ + = ?
None — deactivate all but one (at least one must remain).

Toggle individual chips to fine-tune. At least one symbol must remain active when the Symbols count is non-zero.
Validation rules
The generator validates before producing a password: the sum of all minimums must not exceed the total length, and the sum of all maximums must not be less than the total length. An error notification is shown if either condition fails.

Words

Combines random words from a built-in wordlist with separators, affixes, capitalization, and leet-speak substitutions.

Example output (4 words, Title Case, $ separator)
Flame$River$Anchor$Cloud
Options
Number of Words 2 – 20
How many words are drawn from the wordlist and joined together.
Word Separator
The character or characters placed between each word. See the Shared Options — Separator section for the full list of choices.
Capitalization
Controls how each word's letters are cased. See Shared Options — Capitalization.
Prefix & Suffix
Optional characters prepended (prefix) or appended (suffix) to the full password. See Shared Options — Prefix & Suffix.
Leet Speak
Substitutes selected letters with look-alike characters. See Shared Options — Leet Speak.
Word pills & lock
After generation, each word appears as a clickable pill. Clicking a pill swaps just that word while keeping all others and the current separator/affix values — useful for quickly replacing one word you don't like without regenerating the whole password.

The lock icon beside the pills freezes the current prefix, separator, and suffix so that clicking a word pill or the "Generate" button replaces only the words without re-rolling the affixes.
Character count pill
Once a password is generated, a small pill badge appears inside the output field to the left of the copy button, showing the total character count of the password at a glance. The pill is hidden when the field is empty.

Passphrase

Builds a phrase from a configurable sequence of word slots, each with a grammar role (adjective, adverb, noun, verb) and an optional thematic category.

Example output (adj + noun + verb, UPPERCASE, $ separator)
SILENT$RIVER$FLOWS
Word Slots
The passphrase is assembled from an ordered sequence of word slots (up to 8). Each slot has a type and a category:

TypeColorAvailable Categories
Adj Blue Random, Colors, Size, Texture, Mood, Weather, Time
Adv Teal Random, Manner, Intensity, Time, Place
Noun Slate Random, Animals, Vehicles, Food, Places, Nature, Tech, Jobs
Verb Amber Random, Movement, Action, Nature, Cognition

Each category draws from a curated list of ~100–220 words per category (2,400+ words total across all categories). The word pool is large enough that repeated words are rare even across many generations.
Slots can be added, removed, and reordered with the arrow buttons. The default structure is Adj + Noun + Verb.
Additional options
Word Separator
See Shared Options — Separator.
Capitalization
See Shared Options — Capitalization.
Prefix & Suffix
See Shared Options — Prefix & Suffix.
Leet Speak
See Shared Options — Leet Speak.
Word pills & lock
Same as in Words. Each word pill is color-coded to its slot type.
Character count pill
A badge inside the output field shows the total character length of the generated passphrase. Hidden when the field is empty.

Wireless

A WiFi-optimized passphrase generator with router-friendly defaults and an alliteration mode.

Example output (defaults: Adj + Noun, Title Case, hyphen, 2-digit suffix)
Crimson-River42
Example output with alliteration (letter S)
Silent-Sparrow87

Wireless uses the same slot-based word engine as Passphrase but ships with presets suited for WiFi network names: memorable, typeable on a phone keyboard, and visually distinct. All Passphrase options are available here with independent settings storage.

Generated passwords are guaranteed to be at least 8 characters. If a result falls below this minimum, the generator automatically retries (up to 10 times) before accepting the output.

Default presets
Word slots
Starts with Adj + Noun — a concise, memorable two-word phrase. Additional slots can be added to increase length and variety.
Separator
Hyphen — widely accepted by routers and easy to type on mobile.
Capitalization
Title Case — each word capitalized for readability.
Prefix
None — no prefix by default.
Suffix
2 Random Digits — a short numeric suffix helps meet the 8-character minimum and satisfies many router password requirements without making the phrase hard to remember.
Alliteration mode
The Alliteration toggle appears in the Word Slots card header. When enabled:

  • The generator finds letters that exist in all active slot categories, then picks one at random.
  • Every word pool is filtered to words beginning with that letter before a word is drawn.
  • The active letter is shown as a badge on the toggle button (e.g. S).
  • Clicking a word pill to swap a single word also respects the locked letter.
  • Hitting "Generate WiFi Password" picks a new shared letter each time.
  • If no letter is shared across all categories (rare), the constraint is silently dropped for that slot.

Alliteration produces passwords that are easier to dictate aloud: "Silent Sparrow Storm" is far easier to spell out over a phone than a random mix of words.
All other options
Word Slots
Same as Passphrase. Up to 8 slots; each has a type (Adj, Adv, Noun, Verb) and a thematic category.
Separator, Capitalization, Prefix & Suffix, Leet Speak
Identical to Passphrase. Settings are stored independently under the wifi.* namespace in localStorage.
Word pills & lock
Each generated word appears as a clickable pill. The lock icon freezes prefix/separator/suffix while swapping words. When alliteration is on, swapped words are drawn from the same letter-filtered pool.
Character count pill
A badge inside the output field shows the total character length of the generated password. Hidden when the field is empty.

Mad Lib

Fills a pre-written sentence template with random words, then compresses it into a password.

Example — "The Hero" template, Title Case, - separator
The-Silent-River-Swiftly-Crosses-The-Ancient-Bridge
Template
Choose one of 12 pre-written story templates. Each template is a sentence structure containing typed placeholders (adj, adv, noun, verb) that get filled with random words at generation time.

NameTemplate structure
The HeroThe {adj} {noun} {adv} {verb} the {adj} {noun}
The VillainA {adj} {noun} {adv} {verb} every {noun}
The Quest{noun} {adv} {verb} beyond the {adj} {noun}

Wise ProverbA {adj} {noun} {adv} {verb} alone

Haiku-ish{adj} {noun} {adv} {verb}
Epic Tale{noun} and {noun} {adv} {verb} the {adj} world
MysteryA {noun} {adv} {verb} the {adj} {noun}

Sci-Fi{adj} {noun} {adv} {verb} every {noun}

Literal words in the template (e.g. "the", "beyond", "alone") are included verbatim in the readable phrase preview but are not compressed into the password — only the filled placeholders are joined with the separator.
Word Categories
Each placeholder occurrence in the selected template gets its own category selector. When a template uses the same type more than once (e.g. two adj slots), each is listed separately and can have a different category. Category choices are the same as in Passphrase.
Additional options
Word Separator
Joins the filled words (not the literal template words). See Shared Options — Separator.
Capitalization
Applied to each filled placeholder word. See Shared Options — Capitalization.
Prefix & Suffix

Leet Speak
See Shared Options — Leet Speak.
Readable phrase preview
Shown below the word pills — displays the full human-readable sentence so you can verify or remember the password structure.
Word pills & lock
Same as in Words. Only token words appear as pills; literal template words are not swappable.
Character count pill
A badge inside the output field shows the total character length of the generated password. Hidden when the field is empty.

Numbers

Generates a numeric PIN or code with configurable limits on repeated and sequential digits.

Example output (8 digits, max 2 repeated, max 2 sequential)
47293816
Options
Number of Digits 4 – 32
Total length of the generated numeric code.
Max Repeated Digits 2 – 5
Limits how many times the same digit may appear consecutively. For example, with a limit of 2, 33 is allowed but 333 is not. A value of 5 is effectively unconstrained for most PIN lengths.
Max Sequential Digits 2 – 5
Limits how long an ascending or descending run may be. For example, with a limit of 3, 123 or 987 is allowed but 1234 or 9876 is not. The direction must be consistent — a direction change resets the counter.

Prefix & Suffix

Optional characters added to the very beginning (prefix) or very end (suffix) of the assembled password. Prefix and suffix are independent — either or both can be set.

The prefix shares the same options as the separator (None through Custom). The suffix has two additional options:

Suffix-only optionDescription
Mirror PrefixReverses the characters of the generated prefix and appends them. If the prefix is @7, the suffix becomes 7@.
Mirror Prefix (new digits)Same reversal, but each digit in the mirrored string is replaced with a fresh random digit — symbols are kept. Useful for symmetric-looking passwords where digits differ.

The lock icon in the output area freezes the current prefix, separator, and suffix values so that clicking a word pill regenerates only the words without re-rolling the affixes.

Leet Speak Substitutions

After capitalization and assembly, selected letters are replaced with look-alike symbols or digits. Case is preserved where possible (e.g. A → @, a → @).

SubstitutionCharacter replacedReplacement
a → @a / A@
e → 3e / E3
i → 1i / I1
o → 0o / O0
s → $s / S$
t → +t / T+
l → !l / L!
b → 8b / B8
g → 9g / G9
z → 2z / Z2

All activates every substitution. None clears all substitutions. Individual chips can be toggled independently. Leet speak is applied last, after capitalization.

Generation History

Each generator maintains its own history of recently generated passwords. Clicking any history entry restores that password to the output field. The history strip only appears when there are 2 or more entries.

The History control in the header sets the global maximum number of entries kept per generator:

SettingEffect
Off (0)History is disabled — no entries are stored. Existing history is cleared immediately.
5 / 10 / 20 / 50Keeps up to that many entries. Reducing the limit trims existing history to fit.

History is stored in browser local storage. It persists across sessions but is cleared when you clear site data for this page.

Active Tab

The last generator tab you had open is automatically saved to localStorage (global.activeTab) and restored the next time you visit the app. If no saved tab exists the app defaults to Simple.

Built With

The tools and technologies used to create this app.

Vue 3
The UI is built with Vue 3 using the Composition API and a single-file component approach compiled by Vite. Reactivity, computed properties, and watchers drive all generator logic without any external state-management library.
Vite
Vite handles development serving and production bundling. It compiles the Vue single-file component, processes CSS, and outputs a minimal static bundle that runs entirely in the browser.
Material Design Icons
All interface icons are served from the @mdi/font icon font. Icons are referenced by CSS class name and require no additional JavaScript.
Bolt (AI-assisted development)
Much of this app was built with assistance from Bolt, an AI-powered coding environment. Bolt accelerated feature development, UI design, and documentation authoring throughout the project.
Render
The app is hosted on Render, which serves the static build output. Deployments happen automatically on every push to the main branch via the render.yaml configuration file.
